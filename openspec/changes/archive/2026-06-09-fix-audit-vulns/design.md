# Design: Fix cargo audit vulnerabilities

## Technical Approach

Upgrade sqlx 0.7.4 ‚Üí 0.8.6 with `default-features = false` and explicit features to drop the `any` feature (which pulls in `sqlx-mysql` ‚Üí `rsa`). Create `.cargo/audit.toml` to suppress 7 informational GTK3 advisories (Tauri dependency, no fix available). The app uses runtime-style API only ‚Äî no compile-time macros ‚Äî so the migration is low-risk.

## Architecture Decisions

### Decision: Feature flag explicit set

| Option | Tradeoff | Decision |
|--------|----------|----------|
| Keep `default-features` | Pulls `any` ‚Üí `sqlx-mysql` ‚Üí `rsa` (RUSTSEC-2023-0071) | **Rejected** ‚Äî defeats purpose |
| `default-features = false` + `runtime-tokio`, `sqlite`, `derive` | Drops `rsa` + `sqlx-postgres`; `derive` provides `FromRow` without compile-time macros | **Chosen** |
| Drop `derive`, add `macros` | `macros` pulls compile-time checker deps unused by this app | **Rejected** ‚Äî `derive` is lighter |

In sqlx 0.8, `macros` was split: `derive` provides `FromRow`/`Type`/`Encode`/`Decode` derives without the compile-time query checking machinery. The app uses `#[derive(sqlx::FromRow)]` in 5 structs ‚Äî this is exactly what `derive` enables.

### Decision: `.cargo/audit.toml` format

| Option | Tradeoff | Decision |
|--------|----------|----------|
| `ignore = ["RUSTSEC-..."]` | Simple, flat list | **Chosen** ‚Äî minimal config |
| Per-advisory `criteria` blocks | Over-engineered for known-safe infos | Rejected |

GTK3 advisories are informational ‚Äî Tauri requires GTK3 on Linux, no patch available. List TBD by running `cargo audit` during apply.

## Code Impact Analysis

| Area | sqlx 0.7 usage | 0.8 impact |
|------|----------------|------------|
| `sqlx::Error::PoolClosed` | Used in `lib.rs` test (line 163) | **Variant exists in 0.8** ‚Äî no change |
| `#[derive(sqlx::FromRow)]` | 5 structs in `export_service.rs`, `search.rs` | **Unchanged** ‚Äî `derive` feature provides it |
| `sqlx::query(...)` | 40+ calls across codebase | **Signature unchanged** ‚Äî runtime API |
| `sqlx::query_as::<_, T>(...)` | In `export_service.rs`, `migrations/mod.rs` | **Signature unchanged** |
| `sqlx::query_scalar::<_, T>(...)` | In `lib.rs`, `sync.rs`, `migrations/mod.rs`, etc. | **Signature unchanged** |
| `SqlitePoolOptions::new().connect(url)` | `lib.rs` line 36-38 | **Unchanged** |
| `SqlitePool::connect("sqlite::memory:")` | 10 test helpers | **Unchanged** ‚Äî `Pool::connect(&str)` still works |
| `From<sqlx::Error> for AppError` | `lib.rs` line 108-111 | **Unchanged** ‚Äî `sqlx::Error` enum is non-exhaustive but `Display` impl unchanged |
| `is_table_not_found` | `migrations/mod.rs` line 313 ‚Äî string match on `err.to_string()` | **Unchanged** ‚Äî error message format stable |

**No code changes needed** for sqlx 0.8 API compatibility. The only file modifications are `Cargo.toml` (version + features) and potentially zero `.rs` changes.

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Compile | Full workspace build | `cargo build` ‚Äî catches any API break first |
| Unit | All existing tests | `cargo test` ‚Äî covers sqlx-dependent code paths (migrations, settings, sync, export, search) |
| Audit | Advisory resolution | `cargo audit` ‚Äî verify RUSTSEC-2023-0071 + RUSTSEC-2024-0363 are gone, GTK3 advisories show as `[ignored]` |

### Test order
1. `cargo build` (fastest feedback)
2. `cargo test` (functional correctness)
3. `cargo audit` (security verification)

## Migration / Rollout

No migration required. Cargo handles the dependency tree atomically.

**Rollback**: revert `Cargo.toml`, delete `.cargo/audit.toml`, run `cargo update` to restore lockfile. Covers all 3 changed areas.
